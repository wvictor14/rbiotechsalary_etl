# Serving DuckDB Marts to GitHub Pages

This project uses **DuckDB Wasm + Parquet on GitHub Pages** to serve salary data analytics to the web.

## How It Works

1. **dbt builds your marts** (scheduled weekly) and materializes them in `dev.duckdb`
2. **GitHub Action exports Parquet files** (`export_parquet.py`) from each table
3. **Parquet files are deployed to GitHub Pages** as static files (`docs/data/`)
4. **Pure HTML dashboard** (`docs/index.html`) loads DuckDB Wasm in the browser
5. **DuckDB Wasm reads Parquet directly from CDN**, no backend needed
6. **Users query in the browser** with full SQL support

## Setup

### 1. Enable GitHub Pages

Go to your GitHub repo → Settings → Pages:
- Source: Deploy from a branch
- Branch: `main` (or your default)
- Folder: `/docs`

Click Save. GitHub will publish to `https://YOUR_USERNAME.github.io/rbiotechsalary_etl/`

### 2. Configure GitHub Actions (Optional)

The workflow `.github/workflows/export-parquet.yml` runs weekly to export fresh Parquet files. It uses the `GITHUB_TOKEN` automatically, so no secret setup needed.

To trigger manually:
1. Go to Actions tab
2. Click "Export Parquet Marts"
3. Click "Run workflow"

To change the schedule, edit the cron in `export-parquet.yml`. Current: Sunday 2 AM UTC.

### 3. Test Locally

```bash
# Export Parquet files to docs/data/
python export_parquet.py

# Open docs/index.html in your browser
open docs/index.html
```

Then:
1. Select a table from the dropdown (e.g., "Salary Facts")
2. Click Query
3. Results appear in the table

## Files

- `docs/index.html` — Pure HTML dashboard (DuckDB Wasm + UI)
- `docs/data/` — Parquet export directory (generated by export_parquet.py or GitHub Action)
- `export_parquet.py` — Script to export all DuckDB tables as Parquet
- `.github/workflows/export-parquet.yml` — GitHub Action to rebuild weekly

## Notes

### Data Freshness

- GitHub Action runs **Sunday 2 AM UTC**
- Data is cached in browser; refresh to reload
- Parquet files are versioned implicitly (one deployment per run)

### Table Selection

The dashboard includes all public-facing tables:
- `fct_salaries` — Fact table (main data)
- `dim_companies` — Company dimensions
- `dim_jobs` — Job dimensions
- `dim_locations` — Location dimensions
- `dim_job_titles` — Job title mappings

(Seed/reference tables like `country_names`, `city_names` also exported but hidden from UI)

### Queries

Full SQL in DuckDB syntax. Examples:
```sql
SELECT company, AVG(salary) as avg_salary FROM fct_salaries GROUP BY company LIMIT 10
SELECT * FROM fct_salaries WHERE salary > 100000
```

### Limitations

- **Read-only** — Data cannot be modified from the dashboard
- **Size** — Parquet files must fit in browser memory (9 MB total is fine)
- **Latency** — First load downloads Parquet; subsequent queries are instant (in-memory)
- **CORS** — Works via CDN, no CORS issues

### Future Enhancements

- Add charting (Plotly, Observable)
- Pre-compute aggregations in dbt as separate marts
- Export separate Parquet per "dashboard" (e.g., `exec_summary.parquet`)
- Add download button to export filtered results as CSV

## Troubleshooting

**Dashboard won't load:**
- Check browser console (F12) for errors
- Verify Parquet files exist: `ls docs/data/`
- Verify GitHub Pages is enabled (Settings → Pages)

**Data is stale:**
- Manually trigger the workflow (Actions tab)
- Or wait for next Sunday 2 AM UTC run

**Table dropdown is empty:**
- Make sure you've run `python export_parquet.py` once
- Check `docs/data/` exists and has .parquet files

**Query fails:**
- Check table name (case-sensitive)
- Verify table exists: `duckdb dev.duckdb "SHOW TABLES;"`
